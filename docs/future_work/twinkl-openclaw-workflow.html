KEY INSIGHT → The phone never runs inference, stores models, or processes data. It's literally just a chat window. OpenClaw on laptop/server handles all orchestration. Heavy ML inference is offloaded to external APIs. The only "compute" on the user's device is rendering text messages.
📱 Phone / Messaging App NEAR ZERO
Thin client only — sends/receives text via existing chat APIs. No ML, no storage, no processing.
🖥 OpenClaw Gateway LIGHT → MODERATE
Node.js process handling message routing, heartbeat scheduling, skill orchestration, and memory I/O. CPU-bound only during skill execution. ~200-500MB RAM typical.
🧭 Twinkl Skill Package LIGHT
Markdown-defined logic for onboarding (BWS), nudge classification, digest generation. Most "intelligence" is delegated to the LLM API — the skill just orchestrates.
☁️ LLM API (External) OFFLOADED ENTIRELY
All heavy inference (VIF Critic scoring, Coach RAG generation, nudge phrasing) hits the external API. This is the most expensive compute — but it's someone else's GPU cluster.
📐 VIF Critic Model LIGHT (IF LOCAL) / NONE (IF API)
If hosted locally as a fine-tuned model: moderate GPU spike during scoring. If exposed as API endpoint: zero local compute. For capstone POC, API is the move.
💾 Local Storage NEGLIGIBLE
MEMORY.md, daily logs, parquet/SQLite for Twinkl state. All sequential disk I/O. A Raspberry Pi could handle this.
✓ VERDICT: NO COMPUTE BOTTLENECK
Your instinct is correct. The phone is a dumb pipe. OpenClaw on a modern laptop (your M4 Pro is overkill) handles orchestration comfortably. All heavy inference is API-offloaded. The only scaling concern is token cost from heartbeat + multi-signal sensing hitting the LLM API — that's a dollars problem, not a compute problem.
💻 Local Laptop
Heartbeat only runs while laptop is open. Privacy maximum — nothing leaves your machine except LLM API calls. Perfect for capstone demo. M4 Pro could even run a local VIF Critic.
🖥 Server (VPS/Cloud)
Heartbeat runs 24/7 — catches late-night emails, weekend calendar events. Required for production multi-user. Adds ~$5-20/mo hosting but architecture is identical.
