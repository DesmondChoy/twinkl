VIF Evaluation Pipeline

Each downstream evaluation depends on the reliability of the stage before it.

Stage 1

Data Preparation

Judge-generated supervision and human calibration establish the training target.

Evaluation

Judge Validation

Checks whether labels are consistent enough to train on.

Stage 2

Training

The Critic learns ordinal alignment signals from the curated dataset.

Evaluation

Value Modeling

Measures whether the student recovers useful per-entry alignment signal.

Stage 3

Inference

Runtime scoring, uncertainty, and weekly aggregation produce drift-ready signals.

Evaluation

Drift Detection

Tests whether crash/rut routing behaves correctly on weekly patterns.

Stage 4

User Output

Coach artifacts translate the weekly signal into readable, grounded feedback.

Evaluation

Explanation Quality

Assesses whether feedback is useful, grounded, and understandable.
